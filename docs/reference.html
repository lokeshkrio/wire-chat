Technical Reference & Protocols

Specifications & schemas

Formal definitions of WireChat's JSON payload protocol, SQLite database tables, and customizable configuration properties.

WebSocket JSON Protocol

A detailed mapping of every packet type sent back and forth between client and server.

AUTH

Authentication Packets

Clients must authenticate before sending chat messages or performing commands.

Client Login Request
{
  "type": "login",
  "username": "alice",
  "password": "secret"
}
Server Auth Success Response
{
  "type": "auth_success",
  "username": "alice"
}
CHAT

Messaging Packets

Used for client messaging inside channels and private Direct Messages (DMs).

Client Chat Message
{
  "type": "message",
  "content": "Hello tech room!"
}
Server Chat Broadcast
{
  "type": "message",
  "username": "alice",
  "room": "tech",
  "content": "Hello tech room!",
  "timestamp": 1715901243100
}
DATA

Metadata & Query Packets

Emitted in response to command queries like `/rooms` or `/history`.

Server Room List Response
{
  "type": "rooms_list",
  "rooms": [
    {
      "name": "general",
      "description": "General",
      "users": 4
    }
  ]
}
Server Chat History Response
{
  "type": "history",
  "messages": [
    {
      "username": "alice",
      "room": "tech",
      "content": "Hey there",
      "timestamp": 1715901243000
    }
  ]
}

SQLite Database Schema

Structure of the database tables created automatically in chat.db.

SQL Table Definitions

-- Table: users (Stores user account credentials)
CREATE TABLE IF NOT EXISTS users (
  username TEXT PRIMARY KEY,
  password TEXT -- Argon2id secure hash
);

-- Table: messages (Stores chat room histories)
CREATE TABLE IF NOT EXISTS messages (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  room TEXT,
  sender TEXT,
  content TEXT,
  timestamp INTEGER -- Unix Epoch milliseconds
);

Configuration Guide

Modify parameters to adjust network settings, database locations, or styling defaults.

Setting / Variable Default Value Description / Path
port 5051 Defined inside server.js when initiating the WebSocketServer.
DatabaseSync Path "../../../chat.db" Resolved inside database.js relative to its directory in server/modules/services/.
Argon2 Hashing Type Argon2id Controlled via defaults from the `argon2` module in database.js.
TUI Refresh Mode smartCSR: true Specified inside client.js when building the screen to optimize rendering.
History Limit 50 The maximum messages fetched when /history is executed without args.
